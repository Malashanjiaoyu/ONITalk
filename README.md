# ONITalk

《缺氧》（Oxygen Not Included）的上下文感知复制人对话模组。复制人会根据闲聊、压力、呼吸和环境状态产生短台词；没有远程 API 时自动使用内置离线台词。

## 长期记忆管理

0.9.0 在聊天窗口标题栏增加“库”按钮：

- 查看当前殖民地的关系、累计台词、行动和重大事件数量；
- 浏览每类最新摘要，完整数据不因摘要数量限制而丢失；
- 导出完整 schema 3 JSON 到配置目录的 `exports` 文件夹；
- 导出文件不含 API Key，但可能包含游戏生成的台词，请自行保管；
- 清空需要两次确认，会立即写入磁盘，但不会清除当前聊天窗口历史。

阶段 8 计划支持 DeepSeek、OpenAI、OpenRouter、Gemini、SiliconFlow、Ollama、自定义兼容接口、阿里云百炼／通义千问、Groq 和 Anthropic Claude。详细范围见 [阶段 8 规划](docs/STAGE8_PLAN.md)。

## 多服务商配置与验证

0.8.0 将 DeepSeek 专用选项升级为统一配置面板：

- 可选择 DeepSeek、OpenAI、OpenRouter、Gemini、SiliconFlow、Ollama、阿里云百炼、Groq、Claude、离线台词或自定义兼容接口；
- 选择预设后自动填写推荐 Endpoint 和模型，两者仍可手动修改；
- 自定义接口适用于 Kimi、GLM、MiniMax、LM Studio 等 Chat Completions 兼容服务；
- “验证配置”只发送一次最小请求，显示返回模型、耗时或可读错误，不产生游戏台词和记忆；
- Ollama 可不填写真实 Key，云端服务会检查必要的鉴权配置。

服务商和模型可能随时间变化，因此预设只提供推荐值，不锁死用户配置。Anthropic 当前使用其 OpenAI 兼容层；若后续需要原生 Prompt Caching，再增加 Messages API 适配。

## 智能记忆注入

0.7.0 不再机械地取“最近几条”，而是从现有事实记忆中选择更适合当前话题的内容：

- 综合当前触发原因、双方姓名、当前工作、环境和上一句对话计算本地相关性；
- 同时考虑记忆新旧、重要性、参与者和类型，不让单一高频行动占满提示词；
- 提供轻量、平衡、丰富和自定义预算，控制每次请求携带的记忆量；
- 点击聊天窗口标题栏的“忆”，可查看上一次请求实际选中的记忆及评分明细；
- 整个选择过程不调用额外模型，不增加 API 请求次数。

## 行动与重大事件记忆

0.6.0 开始记录游戏已经确认发生的事实：

- 个人行动：建造、修理、挖掘、拆除、收获、制作、研究、治疗、生物照料、设备调校和探索；
- 殖民地事件：科技完成、复制人死亡；
- 相同工作在短时间内重复完成时自动合并为计数记忆；
- 睡觉、如厕、开门等高频低价值行为不会记录；
- 对话生成时可分别读取双方近期行动和殖民地事件。

行动记忆由游戏事件直接生成，不需要额外 API 请求。旧版殖民地记忆文件会在下一次保存时自动升级。

0.7.1 进一步区分完整设施与零散施工：单块砖、梯子、线缆和管道只在后台累计，同一复制人连续完成 4 次后才形成低权重工程摘要；重复次数再多也不会超过 `0.45`。旧存档中的常见零散施工记录会自动重新分类。

## 双向连续对话

0.5.0 将原来的独立单句扩展为受控对话会话：

- 默认由说话者发言、听者回应，共 2 句；最多可设置为 3 句；
- 回复者会读取对方刚说的话、双方最新状态和长期关系记忆；
- 回复前检查双方仍然有效且没有离开交谈范围；
- 全殖民地同时只生成一个会话，API 请求严格串行；
- 可调整听者回应开关、回应概率、最大台词数和回应间隔。

连续对话会增加 API 调用次数。使用高频测试模式时尤其需要留意调用频率。

## 复制人长期记忆

0.4.0 为每对复制人保存有限的共同对话上下文：

- 按游戏提供的殖民地 GUID 分文件保存，不同存档互不混用；
- 记录累计台词数、上次交谈周期和最近若干句共同对话；
- 根据交谈量提供保守的熟悉度线索，不直接虚构好感或敌对数值；
- 可在模组选项关闭长期记忆，或调整每对记忆条数与最大关系数量；
- 切换存档时丢弃旧存档尚未完成的 API 结果。

记忆文件位于 `ONITalk.json` 同级目录的 `memory` 文件夹。它不包含 API Key，但可能包含游戏中生成的台词；聊天窗口本身仍只显示本次游戏会话。

## 聊天历史窗口

0.3.0 在游戏界面增加殖民地对话历史：

- 新台词实时追加并自动滚动；
- 可拖动、从右下角缩放、最小化和清空；
- 位置与尺寸按屏幕比例保存；
- 可在模组选项调整显示开关、消息上限、字号和不透明度；
- 头顶气泡继续保留。

历史仅保留在当前游戏会话内；窗口布局保存在 ONITalk 配置文件中。

## DeepSeek 设置

在游戏主菜单打开“模组”，找到 ONITalk 后点击“选项”。可配置：

- 是否使用 DeepSeek；
- API 接口：默认 `https://api.deepseek.com/chat/completions`；
- 模型：默认 `deepseek-v4-flash`；
- API Key（输入时遮挡）；
- 界面与台词语言、冷却、普通/测试频率和高级生成参数；台词自动跟随界面语言。

配置保存在：

```text
文档\Klei\OxygenNotIncluded\mods\config\ONITalk\ONITalk.json
```

API Key 不会写入游戏日志，但会以明文保存在上述本机 JSON 文件中。不要把该文件上传到 Git、聊天或模组发布包。

如果未填 Key、请求超时、并发请求占用或服务返回错误，ONITalk 会继续显示离线台词，不阻塞游戏模拟。DeepSeek 请求同时最多一个，模组重载时会取消未完成请求。

## 构建

需要 Windows、PowerShell 5.1+、.NET 8 SDK 和本机《缺氧》游戏文件。

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\doctor.ps1
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\build.ps1
```

构建会运行核心检查，并使用本机当前游戏 DLL 编译 `artifacts\ONITalk\ONITalk.dll`。

公开版本变化统一维护在 [CHANGELOG.md](CHANGELOG.md)，发布 GitHub Release 或更新 Steam 创意工坊详情时以该文件为准。

## 本地安装

请先关闭游戏，再运行：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\install-local.ps1
```

安装脚本会逐文件复制并校验哈希。PLib Core/UI/Options 4.25.0.0 以 MIT 许可内置在 ONITalk 中，许可和修改说明随发布文件提供。
