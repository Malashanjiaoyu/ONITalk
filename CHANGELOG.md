## 0.9.0 - 2026-06-29

### 长期记忆管理

- 聊天窗口标题栏新增“库”按钮，可打开当前殖民地的长期记忆管理器。
- 面板显示殖民地名称、ID、关系数、累计台词、行动数和事件数，并列出最新关系、行动与事件摘要。
- 摘要限制每类显示数量，避免长期殖民地一次渲染过多文本；完整数据仍保留在导出文件中。
- “刷新”会重新读取当前内存状态，不需要关闭面板。

### 导出与安全清空

- 可将当前殖民地的完整 schema 3 记忆导出为带时间戳的 JSON，文件不包含 API Key。
- 导出目录位于 ONITalk 配置目录下的 `exports` 文件夹，不改动正在使用的记忆文件。
- 清空长期记忆需要连续两次确认，并明确提示建议先导出备份。
- 清空会取消进行中的对话请求、使旧请求结果失效并立即覆盖磁盘记忆；当前聊天窗口历史保持不变。
- 记忆注入预览与长期记忆管理器互斥打开，避免多个大窗口遮挡游戏。

## 0.7.1 - 2026-06-29

### 记忆精度

- 将砖块、梯子、线缆和管道等拖拽式施工识别为低价值日常工程，不再与完整设施共用 `0.75` 权重。

- 记忆容量不足时优先淘汰尚未晋升的零散施工，保护关系、设施和重大事件记录。

### 兼容性

- 殖民地记忆升级为 schema 3，新增日常工程标记，并继续兼容 schema 1 和 schema 2。
- 旧存档中常见砖块、梯子、线缆、管道和轨道记录会在读取时自动识别并重新计算低权重。
- Core 检查 52/52 通过；0.7.1 Release 构建 0 警告、0 错误。
