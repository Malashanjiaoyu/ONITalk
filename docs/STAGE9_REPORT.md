# 阶段 9：长期记忆管理

状态：完成并通过真实游戏验证。

## 入口与摘要

聊天窗口新增“库”按钮，与“忆”注入预览分别承担长期资料管理和单次请求解释。两个面板互斥打开。

管理器展示：

- 当前殖民地名称与稳定 ID；
- 关系对数和累计对话台词数；
- 行动与殖民地事件总数；
- 每对关系的最新一句、总台词数和最后周期；
- 行动摘要、重要性与日常工程标记；
- 事件周期、内容与重要性。

每类默认最多渲染 30 项，超出部分显示省略数量。该限制只影响 UI，导出仍包含全部数据。

## 完整导出

- 导出当前内存中的完整关系、行动和事件快照。
- 使用 schema 3、缩进 JSON 和毫秒级时间戳文件名。
- 写入 `mods/config/ONITalk/exports`，不覆盖运行中的 `memory` 文件。
- 导出文档不引用 ONITalk 配置对象，因此不可能包含 API Key。

## 安全清空

- 第一次确认说明清空范围、聊天窗口不受影响并建议先导出。
- 第二次确认明确操作会覆盖磁盘且无法撤销。
- 确认后取消当前对话会话并提升记忆世代号，已发出的旧请求不能回写。
- 清空关系、行动、事件和上次注入预览，随后立即保存空的 schema 3 文档。
- 当前会话聊天历史单独管理，不随长期记忆删除。

## 游戏内验收

1. 进入已有记忆的殖民地，点击聊天窗口“库”。
2. 确认摘要中的关系、行动和事件数量与现有存档一致。
3. 点击“刷新”，确认面板保持正常。
4. 点击“导出 JSON”，确认提示成功且文件存在、schema 为 3、没有 `apiKey` 字段。
5. 点击“清空长期记忆”，确认出现两层提示；首轮验收在第二层选择取消，不删除现有记忆。
6. 检查 Player.log 中没有 UI、导出、记忆或 Harmony 错误。
