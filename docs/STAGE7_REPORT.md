# 阶段 7：智能记忆检索与注入预览

状态：完成并通过真实游戏验证。

## 本阶段目标

- 从已有事实记忆中优先找出与当前话题有关的经历，而非只取最新记录。
- 统一控制关系、双方行动和殖民地事件占用的提示词空间。
- 让玩家能在游戏内看到实际注入了哪些内容以及为什么被选中。
- 不增加额外 API 请求。

## 本地评分

每条候选记忆按以下权重计算：

- 当前上下文相关性：35%；
- 时间新近程度：20%；
- 事件重要性：20%；
- 当前参与者关联：15%；
- 记忆类型优先级：10%。

中文相关性使用本地双字片段，英文和数字使用本地关键词。该实现不依赖向量数据库、Embedding 或外部服务。

## 预算和配额

- 轻量：约 250 Token；
- 平衡：约 520 Token，默认；
- 丰富：约 850 Token；
- 自定义：100–1600 Token。

关系对话、说话者行动、听者行动和殖民地事件各有独立上限，并在预算允许时先获得一次公平选择机会，然后再按总分填充剩余空间。

## 游戏内预览

聊天窗口标题栏新增“忆”按钮。预览展示上一次生成使用的：

- 对话上下文和总预算；
- 候选数、选中数和 Token 估算；
- 每条记忆的总分；
- 相关性、时间、重要性、人物和类型分项。

预览只显示本地记忆选择，不包含 API Key。

## 自动验证

- Core 检查：46/46 通过。
- Release 构建：0 警告、0 错误。
- 覆盖：相关旧记忆压过无关新记忆、四类公平选择、预算硬上限、中文/英文混合 Token 估算、预览文本和最终提示词注入。

## 游戏内验收清单

1. 进入已有行动记忆的存档并等待一次 ONITalk 对话。
2. 点击聊天窗口标题栏“忆”，确认预览窗口出现且可以滚动、拖动和关闭。
3. 确认预算、候选数、选中数和评分可读。
4. 确认至少一条显示的内容来自真实关系、行动或殖民地记忆。
5. 检查日志中没有 UI、配置、记忆或 API 错误。

## 实测结果

- 0.7.0 已在现有殖民地存档加载并正常生成对话。
- 聊天窗口“忆”入口及记忆注入预览可正常使用。
- 实测同时暴露了下一项精度优化：普通单块建筑目前沿用通用建造权重，后续应将细小重复施工聚合为低权重工程摘要，避免挤占高价值记忆。
