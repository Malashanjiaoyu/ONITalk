# ONITalk 剩余工作与工作量评估

更新时间：2026-07-06

当前开发版本：`0.10.0`

## 已完成基线

- 阶段 0–9 已实现并通过真实游戏验收。
- 当前包括触发系统、离线回退、DeepSeek 与多服务商配置、聊天窗口、双向对话、长期关系记忆、行动与事件记忆、智能检索、注入预览和长期记忆管理。
- Core 自动检查 71 项通过；0.10.0 Release 构建目标为 0 警告、0 错误。
- 当前本地模组可正常游玩，但还不是完整的公开发布包。

完成度估计：

- 相对原始核心功能：约 90–95%；
- 相对最终 1.0 目标（3 种语言、自定义提示词、GitHub、Steam）：约 88–92%。

## 1. 阶段 10：全模组本地化

状态：已完成；英语、简中和西语的 206 个键已通过自动覆盖检查，界面、台词、运行时窗口、弹窗和记忆预设均已完成游戏内验收。

### 1.1 本地化基础设施

工作量：大。

- 建立稳定字符串键、英语源文本、`.pot` 模板和 `.po` 加载器。
- 增加“跟随游戏语言”和 ONITalk 手动覆盖。
- 配置及记忆只保存稳定语言代码，不保存翻译后的名称。
- 建立缺键、重复键、格式变量和 UTF-8 检查。

剩余英文主要属于模型内部协议、稳定行动分类和存档事实，不属于普通界面外壳。后续本地化重点是长文本布局回归和存量事实的显示策略。

### 1.2 3 种语言文件

工作量：中到大；语言数量已收缩，但全量文本迁移和游戏内布局验收仍不可省略。

- 英语、简体中文、西班牙语。

三种最终语言都要完成覆盖检查和至少一轮真实游戏布局验收；其他语言不再纳入当前项目范围。

### 1.3 界面语言与台词语言联动

工作量：中。

- 界面默认跟随游戏，也可手动选择。
- 台词输出始终跟随最终应用的界面语言。
- 旧 `language` 配置保留兼容，不再暴露独立输入框。
- 英语、简中和西语离线回退均已提供。

## 2. 默认／自定义提示词

状态：已实现并通过游戏内验收。

工作量：中到大。

- 默认与自定义模式切换。
- 多行提示词编辑器。
- 复制默认、恢复默认、字符上限和空值回退。
- `{language}`、`{maxCharacters}`、`{speaker}`、`{listener}`、`{trigger}` 变量。
- 未知变量校验。
- 不调用 API 的最终提示词预览。
- 核心输出协议继续保护：只输出台词、不泄露系统信息、动态状态和记忆仍自动注入。

该功能必须建立在本地化键和输出语言结构之后，否则默认提示词和编辑界面会返工。

## 3. 发布前技术验收

状态：主要回归完成；跨存档隔离、schema 4 升级、故障回退和 Spaced Out! 真实加载均已通过。

工作量：中到大。

必须补充：

- 两个现有殖民地的独立 GUID、文件名和 schema 3 文档已由自检验证；运行时又增加文档 colonyId 二次核对。
- 在备份或一次性测试存档中真正执行一次“清空长期记忆”，验证磁盘和内存同时清空；
- 除 DeepSeek 外，至少实测一个聚合云接口和一个本地 Ollama／LM Studio 接口；
- 本体与 Spaced Out! 环境加载验证已完成；
- 不同分辨率、UI 缩放和长文本语言布局回归；
- 配置新增 schema 4 升级器及迁移前备份；记忆 schema 1/2/3 保持兼容。
- 断网、无效 Key、429、500、超时和模型不存在统一进入离线回退，并采用 30–300 秒指数退避，等待游戏内故障注入验收。
- 长时间高频模式下检查内存、请求并发、日志和文件上限。

无法要求每个云服务都实测，因为需要对应账号和 Key；可通过请求构造测试加三类代表性真实接口覆盖风险。

工作量：中。

- Git 仓库已初始化为 `main`，尚未首次提交；
- `.gitignore`、`.gitattributes` 和敏感信息扫描已覆盖配置、存档、记忆、日志、内部工作状态和个人路径；
- GitHub Actions 已运行 Core、本地化与敏感信息可移植检查；完整 DLL 构建仍需本机游戏程序集；

- 项目根许可证尚未选择；

CI 自动构建不是绝对首发阻塞项，但建议在公开仓库建立后补上。

- 多语言游戏内校对与布局修复：1–2 个工作日；
