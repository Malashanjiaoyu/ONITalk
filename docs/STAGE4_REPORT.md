# 阶段 4：复制人长期关系记忆

状态：完成并通过真实游戏验证。

## 本阶段目标

- 让同一殖民地中的复制人记得彼此最近说过的话。
- 使用真实存档标识隔离记忆，避免不同殖民地串线。
- 让记忆规模有明确上限，不影响长期档性能。
- 不虚构游戏本身没有提供的精确好感数值。

## 已实现

- 使用 `SaveLoader.Instance.GameInfo.colonyGuid` 作为殖民地记忆主键。
- 每对复制人记录累计台词数、最后周期和最近台词。
- 生成提示词时加入共同经历及保守熟悉度。
- 每 15 秒按需保存，退出或切换存档时立即刷新。
- 写入采用临时文件；旧文件保留为 `.bak`，读取失败时自动回退。
- 每对默认保留 8 句，单殖民地默认最多 200 对关系，均可在模组选项调整。
- 使用记忆世代号隔离未完成请求，避免旧存档 API 响应进入新存档。

## 文件位置

```text
文档\Klei\OxygenNotIncluded\mods\config\ONITalk\memory\<colony-guid>.json
```

文件只保存殖民地元数据和有限的对话关系，不包含 DeepSeek API Key。

## 游戏内验收清单

1. 进入存档并等待至少一条 ONITalk 台词。
2. 确认 `memory` 目录生成当前殖民地 JSON 文件。
3. 退出并重新载入同一存档，确认日志显示载入的关系数大于 0。
4. 继续触发同一对复制人对话，确认累计数和最近台词继续增长。
5. 载入另一殖民地，确认使用不同文件且关系数不继承。
