# 0.7.1：行动记忆降噪

状态：完成并通过真实游戏验证。

## 问题

0.7.0 将所有 `Constructable` 统一赋予 `0.75` 重要性。实测发现普通单块砖也会获得与完整机器接近的叙事地位，并可能借助行动类别保底名额进入提示词。

## 新规则

- 使用游戏 `BuildingDef.DragBuild` 和 `BuildingDef.IsTilePiece` 识别拖拽式、砖块式日常施工，不依赖本地化建筑名称。
- 第 1–3 次只保存为本地聚合计数，不参与记忆注入。
- 第 4 次起形成“完成连续铺设”摘要并参与检索。
- 日常工程重要性从约 `0.21` 开始，随聚合缓慢增加，最高为 `0.45`。
- 非日常功能建筑继续沿用建造重要性并立即参与检索。
- 容量裁剪优先移除未晋升的零散工程。

## 旧存档迁移

记忆文件升级为 schema 3，并继续读取 schema 1/2。旧记录没有日常工程标记，因此迁移时仅对常见砖块、梯子、电线、管道、轨道及其英文名称进行保守识别；命中后按现有计数重新计算权重。

## 自动验证

- Core 检查：52/52 通过。
- Release 构建：0 警告、0 错误。
- 覆盖：单次施工不注入、四次晋升、低权重封顶、功能建筑立即注入、旧砖块记录迁移。

## 游戏内验收

1. 让同一复制人依次建造 1–3 块砖，确认“忆”预览中没有单块施工记忆。
2. 完成第 4 块后触发一次对话，确认预览出现“完成连续铺设”且重要性约为 `0.30`。
3. 建造一个完整功能设施，确认无需累计 4 次即可进入行动候选。
4. 检查殖民地记忆文件升级为 schema 3，日常工程含 `isRoutine: true`。
5. 确认日志没有 Harmony、记忆、UI 或 API 错误。

## 实测结果

- 0.7.1 已成功加载，殖民地记忆文件升级为 schema 3，共恢复并维护 14 条行动记录。
- 奥托完成第 4 块砖时，日志出现“完成连续铺设（砖块），共 4 次”；JSON 权重为 `0.30`。
- 1–3 次砖块和梯子记录权重为 `0.21–0.27`，保留 `isRoutine: true`，未达到晋升门槛。
- 6 次电线聚合后的权重为 `0.36`，证明重复施工按低权重缓慢增长。
- 对话与记忆保存正常，本轮没有 ONITalk 的 Harmony、记忆、UI 或 API 警告与错误。
- Player.log 中另有一条发生在 ONITalk DLL 加载前的本地化异常，堆栈指向 `TranslateFixMod`，与 ONITalk 无关。
