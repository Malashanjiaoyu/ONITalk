# 阶段 10 规划：全模组本地化与提示词配置

状态：实施中；本地化注册、英语源文本、简中翻译与标准选项迁移已完成首批实现。

## 阶段目标

- 让 ONITalk 的选项、聊天窗口、记忆工具、对话框、提示与错误信息完整支持多语言。
- 界面语言默认跟随《缺氧》，同时提供 ONITalk 独立语言覆盖。
- 复制人台词输出语言直接跟随 ONITalk 界面语言，避免重复配置。
- 提供可恢复、可预览、可校验的默认／自定义提示词。

## 首批语言

- English：源文本、通用基线与最终回退语言；
- 简体中文：当前开发与主要验收语言；
- Español：覆盖西班牙及广大拉丁美洲用户。

最终共 3 种语言。繁体中文、法语、德语、意大利语、日语、韩语、巴西葡萄牙语、俄语、波兰语、土耳其语、乌克兰语和荷兰语不纳入当前项目范围。

## 语言设置

### 界面语言

- 跟随游戏（默认）；
- 手动选择上述任一语言；
- 手动覆盖只影响 ONITalk，不改变《缺氧》本体语言；
- 切换后若当前 UI 无法安全热重载，则明确提示重新打开选项或重启游戏。

### 台词输出语言

- 始终跟随 ONITalk 实际应用的界面语言；
- 旧配置中的 `language` 字段保留兼容，但不再显示独立输入控件；
- 远程提示词与离线回退使用同一种语言。

## 本地化结构

- 以英语作为稳定源文本，生成翻译模板。
- 每种语言独立维护 UTF-8 `.po` 文件。
- 所有用户可见字符串改为稳定键；配置文件和记忆文件继续保存稳定 ID，不保存翻译后的枚举名称。
- 回退顺序为：手动语言 → 游戏语言 → English。
- 自动检查每个语言文件是否缺键、重复键、破坏格式变量或遗漏按钮文本。
- 日志中的内部诊断保持稳定英文，真正展示给玩家的消息使用本地化文本。

## 提示词模式

### 默认模式

- 使用 ONITalk 内置提示词；
- 默认提示词按台词输出语言提供本地化版本；
- 保持角色沉浸、只输出台词、长度限制和事实约束。

### 自定义模式

- 多行编辑自定义风格与行为指令；
- 一键复制当前语言的默认提示词；
- 一键恢复默认；
- 自定义内容为空或无效时自动回退默认；
- 设定合理字符上限，防止配置文件和请求无限增长。

### 模板变量

- `{language}`：台词输出语言；
- `{maxCharacters}`：单句字符上限；
- `{speaker}`：说话者；
- `{listener}`：听者；
- `{trigger}`：当前触发原因。

变量缺失不会报错；未知变量会在保存或预览时明确提示。实时状态、记忆、行动、事件和上一句继续由模组自动注入，不要求用户手写模板。

## 提示词预览

- 显示默认或自定义系统提示词的最终展开结果；
- 使用示例角色数据，不发送 API 请求；
- 不显示 API Key；
- 标明动态上下文与长期记忆仍会在真实请求时追加。

## 实施顺序

1. 建立本地化键、英语源模板和加载器。
2. 迁移全部现有用户可见文本，消除业务代码中的硬编码界面语言。
3. 完成 3 种语言文件和覆盖检查。
4. 加入界面语言设置，并让台词自动跟随。
5. 实现默认／自定义提示词编辑、恢复和预览。
6. 用简中、英语和西班牙语完成真实游戏布局验收。

- 默认／自定义提示词编辑器与本地预览；
- `.pot` 模板、`.po` 文件和翻译维护说明；
